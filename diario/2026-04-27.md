# Relatório de Desenvolvimento - Ortho-Check MVP

Este documento resume as iterações de desenvolvimento realizadas durante a semana do dia 27 de Abril de 2026, focadas na criação e aprimoramento da ferramenta Ortho-Check para fiscalização assistida por IA.

## 📅 Segunda-feira, 27 de Abril de 2026

### 🚀 Iteração 1: Fundação do MVP (14:09 - 17:10)
**Objetivo:** Implementação da estrutura base do sistema conforme `PRD-BASE.md`.
- **Backend:** Setup de servidor Express com integração à API da OpenAI (GPT-4o). Implementação do processamento de imagens em memória (Multer) e conversão para Base64.
- **Frontend:** Criação da interface React (Vite) com design system dark premium (Glassmorphism).
- **Funcionalidade:** Upload de duas imagens (referência e atual) para geração de laudo textual de progresso físico.

### 🎨 Iteração 2: Editor de Imagem Three.js (17:13 - 17:25)
**Objetivo:** Integrar ferramenta de desenho de polígonos conforme `PRD-IMAGE-EDITOR.md`.
- **Tecnologia:** Introdução do Three.js via `@react-three/fiber`.
- **Funcionalidade:** Implementação de modal de edição que intercepta o upload. O usuário agora pode desenhar polígonos sobre as imagens para destacar áreas de interesse antes do envio para a IA.
- **Exportação:** Captura do buffer do WebGL para gerar o arquivo final editado.

### 🛠️ Iteração 3: Refinamentos e Correções (17:27 - 17:35)
**Objetivo:** Melhorar a usabilidade e precisão do editor visual.
- **Correção de Aspect Ratio:** Ajuste do plano de fundo para evitar que as imagens ficassem esticadas (implementação de lógica de "contain").
- **UX de Desenho:** Implementação do fechamento automático de polígonos ao clicar no ponto inicial (marcador verde).
- **Estética Visual:** Substituição de linhas WebGL simples por mesh quads para permitir espessura controlada (3px).
- **Clipping:** Correção de overflow no canvas para evitar que elementos de desenho ultrapassassem o cabeçalho/barra de ferramentas.

### 🧠 Iteração 4: Inteligência e Laudos (17:37 - 17:53)
**Objetivo:** Otimizar a análise da IA e a apresentação dos resultados.
- **Prompt Engineering:** Atualização dos comandos da OpenAI para focar a análise **exclusivamente** dentro das áreas delimitadas pelos polígonos.
- **Estimativas de Área:** Inclusão de lógica no prompt para solicitar estimativas de porcentagem de conclusão por tipo de pavimento/superfície.
- **Renderização de Tabelas:** Integração da biblioteca `remark-gfm` no frontend para suportar tabelas Markdown.
- **UI de Relatório:** Estilização premium para tabelas (zebradas, bordas suaves e responsividade).

## 🧪 Análise do Experimento

**O Experimento e Objetivo:** Este projeto consistiu em um experimento prático para avaliar a viabilidade de automatizar a triagem visual de obras de infraestrutura utilizando modelos de visão computacional de larga escala. O objetivo central era validar se o GPT-4o conseguiria, de forma autônoma e precisa, identificar avanços físicos, tipos de pavimentação e estimar áreas concluídas apenas comparando pares de imagens ortográficas.

**Execução:** A execução envolveu o desenvolvimento de uma pipeline completa que permitiu ao fiscal interceptar o upload para delimitar áreas específicas de interesse por meio de polígonos. Esse refinamento visual serviu para "guiar" a atenção da IA, reduzindo o ruído da análise e focando o processamento nos locais de real evolução da obra.

**Resultado e Conclusão:** O resultado final apresentou-se genérico, conforme o esperado para uma abordagem baseada em dados limitados. Concluiu-se que o envio de apenas duas fotos soltas de uma mesma região não oferece contexto suficiente para garantir um laudo técnico de alta fidelidade. Apesar da capacidade avançada de reconhecimento do modelo GPT-4o, a ausência de uma densidade maior de dados (como séries temporais contínuas ou metadados geoespaciais integrados) limita a profundidade da análise, confirmando que a visão computacional isolada ainda requer fontes de dados complementares para aplicações de fiscalização rigorosa.
